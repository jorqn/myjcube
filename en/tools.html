Two different tools

Those two tools are perfectly complementary…

  • For each OLL case, the OLL Trainer knows the formula to apply in order to place the cube in the configuration to solve. The user tells the tool which configurations he wishes to learn or review, and the tool submits one by one the cases to him. The user has just to apply the provided formula on a real cube, and then solve the OLL step. Please note that the provided formulas are volontarily too long in order not to give any clue concerning the solving formula.
  • The Cubeless Trainer submits a picture of the configuration to solve to the user. This one has just to enter the solving formula. This tools concerns as well the OLL phase as the PLL phase.

The first tool helps to train speed and dexterity, while the second one permits to memorize the formulas in a more permanent way. In fact, it seems that memory of the gesture (useful in the first tool) vanishes much more quickly than the knowledge of the text of the formula.

The first tool is dedicated to the OLL phase because it is quite difficult to place the cube in a particular OLL configuration. For the PLL step, the training process is much easier because you often just have to perform the solving formula of a given case to a solved cube to place you in the right configuration.

Compatible with mobile devices.

Those tools have been designed to be handy on mobile devices. A waiting line is indeed a perfect place for reviewing OLL/PLL formulas ;-)

SHATARO… or not

The OLL Trainer is not related to shataro in any way. The scrambling formulas are given in classical convention, but it does not prevent the user from having memorised the solutions in shataro language. Concerning the Cubeless Trainer, it is able to understand both shataro and standard notation.
